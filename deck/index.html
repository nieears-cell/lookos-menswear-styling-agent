Portfolio case / menswear AI product

LookOS

把五行穿搭重构成可调的男装 AI Styling Agent

这不是“今日大吉色”查询，而是一个能读取日程、天气、衣橱和情绪目标，并把它们转成可操作男装方案的 Style Tuner。

AI Product Agent Workflow Data Feedback

01 / product problem

传统五行穿衣只告诉颜色，但真实穿搭要解决场景。

用户不是想知道“今天白色好不好”，而是想知道：今天见客户、晚上吃饭、天气 23°C，我该怎么穿才可信又不僵。

旧入口

今日颜色：白、银、黄、棕、肤。

真实决策

颜色放在哪个单品上？正式到什么程度？晚上要不要松一点？

新机会

把五行降级为解释层，把入口换成日程、场景和用户想呈现的状态。

02 / core product judgment

AI 不该替用户做审美决定，而应该让用户可控地调出今天的自己。

Formality 正式度

决定西装、针织、鞋款和裤型的正式边界。

Presence 存在感

决定黑色面积、对比度、领带或线条强度。

Freshness 清爽度

决定白灰面积、材质轻重和年龄感。

Calendar 场景权重

客户拜访是主场景，晚餐是切换场景。

Five colors 色彩转译

白/银做清晰，棕/肤做亲和，黄只做少量暖意。

核心动作从“看推荐”变成“调状态”。

03 / agent system

从日历到 Look：Agent 把多约束任务变成一套可穿方案。

Calendar Scene Weight Weather Wardrobe Style Tuner Look
Example input 10:00 团队周会 / 14:30 客户拜访 / 19:00 晚餐

系统把客户拜访设为主锚点，用 Reliable Operator 建立可信感；如果用户降低正式度，则切到 After-hours Ease，适合晚餐过渡。

04 / prototype

新版首页：日程简报 + 三轴 Style Tuner + 实时男装预览。

首屏不再是静态结果页，而是一个可以拖动、可以调性格、可以看即时反馈的小程序操作页。

  • 三条滑杆：正式度、存在感、清爽度。
  • 三套 Look：Reliable Operator、Quiet Authority、After-hours Ease。
  • 三层证据：Why、Evidence、Feedback。

05 / controllable output

同一天，不同状态，生成不同男装方案。

默认可信 提高存在感 降低正式度

06 / evidence layer

面试里真正要讲的是：推荐背后的证据链。

1日程锚点

客户拜访权重最高，晚餐是次场景。

2天气约束

杭州阴 23°C，薄外套可穿，避免厚重内搭。

3衣橱命中

基于已有男装单品组合，并暴露缺口。

4五行转译

把颜色从玄学话术改成现代风格 metadata。

07 / data loop & role fit

Badcase 被设计成数据闭环，而不是简单“不喜欢”。

“太凌厉、太普通、偏热、有点土”会落成可归因标签，下一次直接影响推荐参数。

太普通style_distinctiveness_low
偏热weather_constraint_failed
有点土style_modernity_low
